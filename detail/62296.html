jsDelivr CDN 失效临时解决方案

jsDelivr CDN 是一个用于开源项目的免费公共 CDN。在世界各地拥有众多的节点，没有流量限制，可在全球范围内使用.
该项目由Cloudflare、Fastly、StackPath、appfleet、NS1 和DigitalOcean等多家知名公司赞助。

在中国大陆地区是由网宿子公司提供服务。


但是在服务运行中，多多少少会出一点问题，在搜索引擎上随便一搜就能得到很多篇文章。

在今天上午出现了大规模的问题，cdn.jsdelivr.net 不能在大陆地区进行解析，问题原因是 SSL 证书过期。然后把问题反馈给 jsDelivr 官方，经过几个小时的等待，jsDelivr 官方发推解释了问题:在中国大陆地区的 ICP 许可证被吊销了。同时把节点路由调整为 fastly。

在服务不能正常运行期间，在 Github issue 里官方成员给了一点小技巧，把三级域名改成其他服务商就可以运行，虽然速度没有以前那么快。

解决办法：
把 cdn.jsdelivr.net 改为以下几个之一就行。

  • fastly.jsdelivr.net
  • testingcf.jsdelivr.net
  • gcore.jsdelivr.net
  • quantil.jsdelivr.net
