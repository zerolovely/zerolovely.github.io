使用 GlobalSSH 加速境外服务器

对于国内用户来说，连接境外服务器能够流畅地操作市非常重要的，但是因为国内众所周知的网络环境，所以还是有点烦恼的。

这里介绍一个免费的加速服务！

2020年5月19日，官方对服务进行了限制

产品介绍

GlobalSSH 是一款致力于提高跨国远程管理服务器效率的产品，旨在解决因为跨国网络不稳定，通过远程管理服务器时，经常会出现卡顿、连接失败、传输速度较慢等现象。运维研发人员在使用本产品后，可以提高，极大程度的减少卡顿、连接失败的情况发生，提高运维工作的效率。
注：本产品同样适用于 windows 服务器的远程登陆服务

服务地址

登录 ucloud 控制台，点击 全部产品，在搜索框内输入 pathx ，就可以看见一个 全球动态加速 PathX，点击进入即可。

或者直接进入https://console.ucloud.cn/upathx/globalssh

使用教程

进入界面后，点击创建按钮，在弹出的页面里填写相关信息。

服务器IP：填写你的服务器IP
服务器靠近区域：根据自己的境外服务器位置选择靠近该服务器的加速区域
端口：填写你的服务器的 ssh 端口。该加速服务同时支持 linux 和 windows 。linux 默认端口为22，windows 默认的远程桌面端口为3389.

创建成功后就可以看到分配的加速域名了

然后利用远程连接工具连接。
在原来该填IP的地方填写加速域名，其他照旧。

到 GlobalSSH 控制界面点击查看详情就可以看见详细的统计信息以及出口IP。同时为保护你的服务器，还可以添加白名单。

场景限制

  • GlobalSSH产品定位是方便海外云主机运维管理，故禁止了80、443端口来杜绝web服务。

  • 相同源站IP的GlobalSSH实例只能创建一个，多余1个的情况会导致TCP连接失败。

  • 如果您的业务需要支持更多的端口和更大的带宽，欢迎使用PathX （带宽充足可共享，最多支持50个端口）。

  • 如果您需要对同一台云主机访问多个端口，除了PathX方案，可以考虑使用SSH隧道用本地端口代理访问远端服务端口，请确保云主机的sshd配置是否支持端口转发

提醒

目前所有用户都可以使用该加速服务，包括未实名用户！

目前该服务是免费提供！

自产品实例创建之日起，7日内单个实例累计出向流量小于2MB，加速功能停用，停用之后要使用该服务重新配置即可。

加速流量上限官方还未注明，且用且珍惜，不要滥用！
